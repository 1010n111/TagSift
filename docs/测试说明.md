# 测试说明

> 供商店审核人员参考。用户不会看到此信息。

---

## 概述

**AI 智能标签分类** 是一款 Chrome 扩展，自动分析网页内容并调用大语言模型（LLM）生成多维度智能标签，帮助用户管理浏览信息。

---

## 核心功能测试步骤

### 1. 首次使用 — 配置 API

1. 安装扩展后，点击浏览器工具栏的扩展图标
2. 弹窗打开，如果未配置 API，会显示提示"⚙️ 配置 API Key — 去设置"
3. 点击「去设置」或右键扩展图标 →「选项」
4. 在设置页填写：
   - **接口地址**：`https://api.deepseek.com/chat/completions`（或其他 OpenAI 兼容接口）
   - **API Key**：输入有效的 DeepSeek / OpenAI API Key
   - **模型名称**：`deepseek-chat`（或其他模型）
5. 点击「🔍 测试连接」验证配置
6. 点击「💾 保存设置」

### 2. 网页标签生成

1. 访问任意新闻/博客/技术文章页面（如 `https://news.ycombinator.com`）
2. 点击扩展图标
3. 弹窗自动抓取网页标题和正文内容
4. AI 自动分析并生成 5 维度标签：
   - 📂 内容领域（如：科技、教育、金融）
   - 🎯 用途场景（如：学习参考、工作文档）
   - 📊 难度等级（单选：入门/进阶/专业）
   - 🔑 核心关键词（如：AI、JavaScript）
   - 🏷️ 内容属性（如：教程、新闻、分析）
5. 标签以 chips 形式展示，支持编辑：
   - **双击标签** → 编辑文本
   - **点击 ×** → 删除标签
   - **点击 +** → 新增标签
6. 标签自动保存（500ms 防抖），关闭弹窗后再次打开同一页面标签仍在

### 3. 收藏夹管理

1. 在弹窗底部点击「📂 收藏夹」
2. 新标签页打开收藏夹管理页面
3. 左侧面板包含搜索栏和标签统计
4. 右侧显示书签树（默认文件夹折叠，点击展开）
5. **按标签搜索**：在左侧搜索框输入关键词（支持空格分隔多词 AND 搜索）
6. **从标签统计点选**：点击左侧的标签自动填入搜索框

### 4. 批量处理书签

**方式一：手动勾选**
1. 在收藏夹页面，勾选任意书签前的复选框
2. 点击「🚀 批量处理」
3. 扩展逐个打开书签 URL → 提取内容 → 调用 AI → 保存标签

**方式二：全部覆盖**
1. 不勾选复选框，直接点击「🚀 批量处理」
2. 在弹出的对话框中选择「🔄 全部覆盖」
3. 所有书签（含已有标签的）将被重新处理

**方式三：只处理收藏栏**
1. 不勾选复选框，点击「🚀 批量处理」
2. 选择「✅ 只处理收藏栏」
3. 仅处理收藏栏中尚未生成标签的书签
4. 如果收藏栏为空，会提示先添加书签

**处理中操作**：
- 进度条显示当前进度
- 红色「⏹ 结束」按钮可随时中止
- 中止后按钮变为「🚀 继续处理」，可续跑

### 5. 标签搜索

1. 在收藏夹页面左侧搜索框输入关键词
2. 支持**多词搜索**：输入 `科技 入门` 只显示同时包含"科技"和"入门"的书签
3. 搜索结果以扁平列表展示，每条显示完整的文件夹路径
4. 清空输入框恢复树形视图

### 6. 数据导入导出

1. **导出**：点击「📤 导出」下载 JSON 文件
2. **导入**：点击「📥 导入」选择 JSON 文件，同 URL 的记录自动覆盖

### 7. 主题切换

1. 在设置页选择「外观主题」：
   - 浅色模式
   - 深色模式
   - 跟随系统
2. 切换后即时预览

## 权限说明

| 权限 | 用途 |
|------|------|
| `storage` | 本地保存标签和设置 |
| `activeTab` | 读取当前页面内容 |
| `scripting` | 注入内容提取脚本 |
| `bookmarks` | 读取收藏夹用于批量处理 |
| `http://*/` `https://*/` | 批量处理时打开书签 URL |

所有数据仅存储在用户本地 `chrome.storage.local`，不向扩展开发者发送任何数据。

---

## 已知限制

- 批量处理时会在后台打开标签页（可见但不会被聚焦），处理完后自动关闭
- AI 标签质量取决于配置的 LLM 模型能力
- 浏览器内部页面（`chrome://`、`edge://`）无法抓取内容
- 需要网络连接以调用 AI API
