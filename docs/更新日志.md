# 📋 更新日志

> 本文档记录 AI 智能标签分类 Chrome 扩展的所有功能更新。

## v1.2.0 — 搜索优化 + 布局重构

### 🎨 布局重构

#### 1. 收藏夹管理页改为左右结构

- 原上下布局（搜索栏 + 标签统计面板各占一行）改为 **左侧面板 + 右侧内容** 双栏布局
- 左侧固定 260px 面板：顶部搜索栏 + 下方标签统计列表（纵向滚动）
- 右侧主区域：树形视图 / 搜索结果
- 面板始终显示，不再有 `hidden` 切换
- 移除了「🏷️ 标签统计」切换按钮和关闭按钮

#### 2. 搜索栏紧凑化

- 搜索栏 padding 从 `8px 24px` 压缩至 `6px 16px`
- 输入框 padding 从 `7px 12px` 压缩至 `4px 8px`
- 字号从 13px 降至 12px
- 移至左侧面板后不再占用主内容区垂直空间

### 🔍 搜索增强

#### 3. 多标签 AND 搜索

- 搜索框支持空格分隔多个关键词，必须**全部命中**（AND 逻辑）
- 例如：`科技 入门` → 仅显示同时包含"科技"和"入门"的书签
- 从标签统计点击标签时追加到搜索框（自动去重，空格分隔）
- 输入框提示改为「输入标签（空格分隔可多选）…」

### ✅ 收藏栏流程优化

#### 4. 未收藏网页提示收藏

- 弹窗标签区新增收藏提示条：`⭐ 此网页未添加到收藏栏，点此收藏 以便后续批量处理`
- 点击「点此收藏」直接调用 `chrome.bookmarks.create()` 添加到收藏栏
- 收藏成功后自动触发 AI 标签生成
- 重新生成按钮同理 — 未收藏时先提示收藏

#### 5. 「只处理收藏栏」模式

- 弹窗模式名从"只处理剩余的"改为 **「只处理收藏栏」**
- 内部使用 `chrome.bookmarks.getSubTree('1')` 仅获取收藏栏子树
- 收藏栏为空时提示「收藏栏中没有书签」
- 勾选框手动选择不受限 — 任意位置的书签均可单独处理
- 顶部统计信息改为 `收藏栏 3/10 | 全部 15/50` 双指标

### 📤 数据导入导出

#### 6. 导出标签数据

- 收藏夹页面顶部新增「📤 导出」按钮
- 将所有标签记录导出为 JSON 文件（`ai-tags-YYYY-MM-DD.json`）
- 导出格式包含版本号、导出时间、记录总数

#### 7. 导入标签数据

- 收藏夹页面顶部新增「📥 导入」按钮
- 点击后打开文件选择器，仅显示 `.json` 文件
- 导入前显示确认弹窗：`即将导入 N 条标签记录，当前已有 M 条记录`
- 同 URL 的记录自动覆盖（按 URL 去重）
- 导入完成后自动刷新页面展示新数据
- 导入格式验证：检查 `version` 和 `records` 字段合法性

### 🎯 UI / UX 优化

#### 8. 文件夹排序对齐浏览器

- 收藏夹树改为按 Chrome 原生 `index` 字段排序，与浏览器收藏夹管理器的显示顺序一致

#### 9. 文件夹默认折叠

- 打开收藏夹管理页面时，所有文件夹默认收起，页面更清爽

## v1.1.0 — 收藏夹管理 + 搜索统计

### 🆕 新增功能

#### 1. 收藏夹批量标签生成

- 新增独立页面 `bookmarks/bookmarks.html`，展示全部 Chrome 收藏夹树形结构
- 每个书签显示已有标签（✅）或待处理状态（⏳）
- **批量处理**：逐条遍历书签 URL → 开后台 tab → 注入 content 提取正文 → 调 LLM 生成标签 → 保存 → 关 tab
- **模式选择弹窗**：处理前弹出两种模式供选择
  - 🔄 **全部覆盖** — 重新处理所有书签，覆盖已有标签
  - ✅ **只处理剩余的** — 仅处理尚未生成标签的书签
- **⏹ 结束按钮**：红色发光 + 脉冲动画，可随时中止批量处理
  - 中止后按钮变为「🚀 继续处理」，可续跑
- **进度条**：实时显示处理进度（`已处理 / 总数`）
- **入口**：弹窗底部新增「📂 收藏夹」按钮，点击在新标签页打开

#### 2. 按标签搜索书签

- 收藏夹页面顶部搜索栏，输入关键词即搜
- **搜索范围**：标签值（全部 5 维度）+ 书签标题 + 文件夹路径
- **扁平结果列表**：搜索时隐藏树形视图，切换为结果列表
  - 每条结果显示完整文件夹路径 → 书签标题 → URL → 所有标签
- **🔍 200ms 防抖**，输入时不卡顿
- **匹配计数**：显示"匹配 N 个"
- **✕ 清除按钮**：清空输入即恢复树形视图

#### 3. 标签统计面板

- 收藏夹页面顶部新增「🏷️ 标签统计」按钮
- 打开后显示所有标签，按出现次数降序排列
- 每条标签展示：所属维度 + 标签名 + 计数徽章
- **点击即搜**：点击任意标签自动填入搜索框并触发搜索

### ✨ 功能增强

#### 4. 自动保存标签

- AI 生成标签成功后立即静默保存
- 编辑标签（新增/删除/修改）后 500ms 防抖自动写入 `chrome.storage.local`
- 手工「💾 保存」按钮保留，点击时弹出 Toast 确认
- `saveTags(silent)` 支持静默模式参数

#### 5. 主题切换联动

- 设置页新增主题下拉：**浅色模式 / 深色模式 / 跟随系统**
- 使用 `data-theme` 属性 + CSS 变量实现三模式覆盖
- 选项页切换主题时即时预览（存盘后弹窗同步生效）
- 弹窗（popup）、选项页（options）、收藏夹页（bookmarks）三端统一

#### 6. 已有标签缓存

- 打开已保存过的网页时，直接加载已有标签，不再重复调用 AI
- 节省 API 调用次数，提升响应速度

#### 7. 内存优化

- 弹窗关闭时自动清理：暂停 autoSave 定时器 + 释放大文本内容
- `window.addEventListener('unload')` 统一回收

#### 8. API 配置引导

- 标签区新增 `config-hint` 提示条：API 未配置时显示"⚙️ 配置 API Key"
- 错误状态新增「⚙️ 去设置」按钮，点击直接跳转设置页
- `showError(text, showSettings)` 支持传入 settings 按钮开关

### 🔧 其他优化

#### 9. 清理未使用代码

- 移除 `MESSAGE_TYPES` 中未使用的常量

## v1.0.0 — 初始发布

### 核心功能

- Manifest V3 Chrome 扩展架构
- 网页正文智能提取（content script）
- 5 维度 AI 标签生成（内容领域 / 用途场景 / 难度等级 / 核心关键词 / 内容属性）
- 标签编辑：双击修改 / × 删除 / + 新增
- 本地存储：按 URL 保存标签，支持更新覆盖
- 历史记录：按时间倒序查看、点击回填、清空全部
- 设置页（options）：API Key 加密存储、连接测试、开关配置
- 深色模式（`prefers-color-scheme`）
- 抓取节流（30s 内不重复抓取）
- API 错误分类（401/429/timeout/network）
- 零外部依赖，纯原生实现
