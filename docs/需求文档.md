> 版本：v1.0

---

## 1. 产品概述

### 1.1 产品定位

一款轻量级 Chrome 浏览器扩展，利用大语言模型（LLM）对用户当前浏览的网页进行**自动内容分析与智能标签分类**，帮助用户高效管理浏览信息、建立个人知识索引。

### 1.2 目标用户

- 需要频繁查阅、整理网页内容的知识工作者
- 有信息收藏癖、希望建立个人标签体系的用户
- 需要快速对网页内容进行分类归档的学生、研究员、产品经理

### 1.3 核心价值

| 痛点 | 解决方案 |
|------|----------|
| 浏览大量网页，事后难以回忆归类 | AI 自动生成多维度标签，一目了然 |
| 手动打标签费时费力 | 一键 AI 分析 + 标签编辑，秒级完成 |
| 标签体系混乱，无统一标准 | 固定 5 维度标准化标签，粒度一致 |
| 换了设备或浏览器，标签丢失 | Chrome storage.local 持久化，跨设备同步 |

## 2. 功能需求

### 2.1 网页内容抓取

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F-01 | 自动抓取 | 打开弹窗时自动获取当前标签页的标题、URL、正文核心文本 | P0 |
| F-02 | 内容净化 | 自动过滤广告、弹窗、导航栏、页脚等非正文冗余内容 | P0 |
| F-03 | 手动刷新 | 提供刷新按钮，用户可重新抓取当前页面内容 | P1 |
| F-04 | 抓取状态反馈 | 抓取中显示加载动画，抓取完成/失败显示对应状态 | P0 |

### 2.2 AI 大模型标签分类

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F-05 | 多模型适配 | 预留 API 接口，可对接 DeepSeek、GPT、通义千问等主流 LLM | P0 |
| F-06 | 5 维度标签生成 | 按固定维度生成标签：内容领域、用途场景、难度等级、核心关键词、内容属性 | P0 |
| F-07 | 标签标准化 | 单标签 2–6 字，简洁无冗余话术 | P0 |
| F-08 | 重新生成 | 用户可触发重新 AI 生成全部标签 | P1 |
| F-09 | 标签编辑 | 支持手动新增、删除、修改单个标签 | P1 |
| F-10 | 智能去重 | AI 生成的标签自动去重、合并同类标签 | P1 |

**5 维度标签说明：**

| 维度 | 含义 | 示例值 |
|------|------|--------|
| 内容领域 | 网页所属的知识/行业领域 | 科技、教育、金融、医疗、设计 |
| 用途场景 | 该网页适合什么场景使用 | 学习参考、工作文档、休闲阅读 |
| 难度等级 | 内容的专业/难易程度 | 入门、进阶、专业 |
| 核心关键词 | 2-4 个代表性关键词 | AI、机器学习、JavaScript |
| 内容属性 | 网页的类型/体裁 | 教程、新闻、分析、工具、报告 |

### 2.3 插件 UI 交互（弹窗）

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F-11 | 弹窗布局 | 极简现代化 UI，适配 Chrome 弹窗标准尺寸 (400×600) | P0 |
| F-12 | 基础信息区 | 展示网页标题、URL、抓取时间 | P0 |
| F-13 | 标签展示区 | 按 5 维度分组展示标签，每个标签可独立编辑/删除 | P0 |
| F-14 | 操作按钮区 | 包含刷新抓取、重新生成、保存、历史记录等按钮 | P0 |
| F-15 | 动效与反馈 | hover 动效、加载动画、请求失败提示、空状态提示 | P1 |
| F-16 | 深色/浅色模式 | 自动跟随系统主题，支持手动切换 | P1 |

### 2.4 本地数据持久化

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F-17 | 自动保存 | 标签编辑后自动保存至 chrome.storage.local | P0 |
| F-18 | URL 去重 | 以 URL 为唯一标识，重复网页自动覆盖旧标签 | P0 |
| F-19 | 历史记录 | 查看所有历史标签记录，按时间倒序排列 | P1 |
| F-20 | 单条删除 | 在历史列表中删除某条记录 | P1 |
| F-21 | 清空全部 | 一键清空所有历史标签数据（需二次确认） | P1 |

### 2.5 基础配置

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F-22 | API 配置 | 设置页填写大模型 API Key、接口地址、模型名称 | P0 |
| F-23 | 自动开关 | 开关：自动抓取网页、自动 AI 生成标签 | P1 |
| F-24 | 自定义规则 | 自定义标签生成的 prompt 规则/额外指令 | P2 |

### 2.6 异常处理

| ID | 场景 | 表现 |
|----|------|------|
| E-01 | 网络请求失败 | 弹窗内显示友好错误提示，提供重试按钮 |
| E-02 | API Key 无效/未配置 | 引导用户前往设置页配置 API Key |
| E-03 | 接口超时 | 超时提示（默认 15s），支持手动重试 |
| E-04 | 网页无正文内容 | 提示"当前页面无可抓取内容"，不阻塞操作 |
| E-05 | 抓取内容过长 | 自动截断至合适长度（如 8000 tokens），避免 API 浪费 |

## 3. 非功能需求

### 3.1 性能

| ID | 要求 |
|----|------|
| N-01 | 弹窗打开到展示内容 ≤ 1s（不含 API 调用的网络时间） |
| N-02 | 内容抓取节流：同一页面 30s 内不重复抓取 |
| N-03 | 后台 Service Worker 无持续内存占用，用完即释放 |
| N-04 | API 调用加入防抖，避免用户连续点击造成重复请求 |

### 3.2 安全

| ID | 要求 |
|----|------|
| N-05 | 权限最小化：只申请 activeTab、storage、scripting 必要权限 |
| N-06 | API Key 使用 chrome.storage.local 存储，不暴露至页面 DOM |
| N-07 | 内容脚本仅注入当前活动标签页，不持久驻留 |
| N-08 | 跨域请求通过 background service worker 代理，避免 CORS 问题 |

### 3.3 兼容性

| ID | 要求 |
|----|------|
| N-09 | 兼容 Chrome ≥ 100、Edge ≥ 100（基于 Manifest V3） |
| N-10 | 适配 Windows / macOS 双平台 |
| N-11 | 深色/浅色模式通过 CSS prefers-color-scheme 自动适配 |

### 3.4 可维护性

| ID | 要求 |
|----|------|
| N-12 | 函数模块化，每个模块职责单一 |
| N-13 | 关键函数有 JSDoc 注释说明 |
| N-14 | 全局常量集中管理，便于配置修改 |
| N-15 | 代码目录结构清晰，与功能模块一一对应 |

## 4. 数据模型

### 4.1 标签记录 (TagRecord)

```typescript
interface TagRecord {
  /** URL 作为唯一标识 */
  url: string;
  /** 网页标题 */
  title: string;
  /** 抓取时间 ISO 字符串 */
  capturedAt: string;
  /** 上次更新时间 ISO 字符串 */
  updatedAt: string;
  /** AI 生成的标签 */
  tags: {
    contentDomain: string[];    // 内容领域
    usageScenario: string[];    // 用途场景
    difficultyLevel: string[];  // 难度等级
    coreKeywords: string[];     // 核心关键词
    contentAttribute: string[]; // 内容属性
  };
  /** 用户手动修改过的标签（持久化用户编辑） */
  edited: boolean;
}
```

### 4.2 配置 (AppConfig)

```typescript
interface AppConfig {
  apiProvider: 'deepseek' | 'openai' | 'tongyi' | 'custom';
  apiKey: string;           // 加密存储
  apiEndpoint: string;
  modelName: string;
  autoCapture: boolean;     // 自动抓取
  autoTag: boolean;         // 自动生成标签
  customPrompt: string;     // 自定义规则
  theme: 'auto' | 'light' | 'dark';
}
```
